# 从“研究工具原型”到“投资情报图谱产品”

## 1. 当前阶段判断

当前项目已经具备一个可用原型的核心闭环：

- 输入公司名称和时间范围
- 拉取新闻与官方披露
- 对新闻做基础事件抽取与聚类
- 用 AI 生成中文事件总结
- 以图谱方式展示结果

但它本质上仍更接近：

- 被整理过的新闻面板
- 研究辅助工具
- 事件摘要原型

还不算真正的“投资情报图谱产品”。

原因很直接：

- 图谱节点仍主要围绕“新闻聚类事件”
- 关系仍以 `公司 -> 事件 -> 来源` 为主
- 缺少投资研究真正关心的结构化实体
- 缺少“事件对股价/基本面的影响”这一层建模
- 缺少持续追踪、对比、筛选、告警和可信度体系

## 2. 产品目标重定义

如果要进化成“投资情报图谱产品”，目标不应该只是：

输入公司名，看看最近发生了什么新闻

而应该变成：

输入公司名或股票代码，快速识别一段时间内真正影响股价和基本面的关键事件，并把这些事件与披露、人物、合作方、监管方、财务信号、市场反应组织成可追溯的图谱。

这意味着产品输出要从“新闻结果”升级成“投资结论中间层”。

## 3. 需要补的核心能力

### 3.1 数据源升级

当前问题：

- 通用新闻源噪音大
- 财经与股价敏感信息覆盖不够稳
- 官方披露占比不够高

目标数据源结构：

1. 一级源：官方披露
- SEC EDGAR
- 公司 IR 页面
- Press Release
- 交易所公告

2. 二级源：金融新闻
- Finnhub Company News
- Financial Modeling Prep Stock News
- Alpha Vantage News & Sentiment
- Reuters / Bloomberg / WSJ 等高质量媒体入口

3. 三级源：市场数据
- 股价
- 成交量
- 财报日历
- 分析师评级
- 期权异动

落地重点：

- 官方披露优先
- 财经新闻优先于泛新闻
- 新闻和股价数据联动

### 3.2 投资事件 Schema 重建

这是最关键的升级点。

当前问题：

- 事件类型太粗
- 图谱还是“新闻中心型”，不是“投资事件中心型”

目标事件类型建议：

- 财报业绩
- 指引上调/下调
- 并购交易
- 合作签约
- 裁员重组
- 监管调查
- 诉讼仲裁
- 回购分红
- 融资发债
- 高管变动
- 产品发布
- 大客户订单
- 供应链变化
- 分析师评级变化
- 股价异动

每个事件需要统一字段：

- `event_type`
- `event_title`
- `event_summary`
- `event_date`
- `actors`
- `counterparties`
- `source_type`
- `evidence`
- `officialness`
- `impact_direction`
- `impact_level`
- `confidence`
- `price_sensitive`

### 3.3 图谱建模升级

当前图谱：

- `Company -> Event -> Source`

目标图谱：

- `Company`
- `Event`
- `Filing`
- `Source`
- `Executive`
- `Counterparty`
- `Product`
- `Regulator`
- `MarketSignal`

推荐关系：

- `ANNOUNCED`
- `FILED`
- `REPORTED_BY`
- `PARTNERED_WITH`
- `ACQUIRED`
- `APPOINTED`
- `CUT_JOBS`
- `INVESTIGATED_BY`
- `AFFECTED_PRICE`
- `MENTIONED_IN`

升级后的图谱要能回答：

- 哪些事件可能影响股价
- 哪些是官方确认，哪些只是媒体报道
- 哪条事件主线持续时间最长
- 哪些事件与财务、监管、管理层变化相关

### 3.4 AI 抽取能力升级

当前问题：

- 更多是“总结新闻”
- 还不是“提取投资结构”

目标能力：

1. 事件分类
- 判断是否属于投资相关事件

2. 实体抽取
- 公司
- 高管
- 合作方
- 监管机构
- 产品
- 地域

3. 影响判断
- 利多 / 利空 / 中性
- 短期 / 中期 / 长期影响

4. 证据绑定
- 每个结论必须对应原文证据

5. 去重与聚合
- 同一事件多来源报道合并

AI 不应只生成“摘要”，而应生成结构化输出。

### 3.5 市场反应层

如果没有市场反应层，这个产品仍然会更像新闻工具，而不是投资工具。

要补的数据和能力：

- 事件发生前后股价变化
- 成交量变化
- 财报日与非财报日区分
- 新闻发布时间与盘前/盘中/盘后关系
- 是否伴随评级变化

目标效果：

- 不是只说“发生了什么”
- 而是能说“发生了什么，以及市场怎么反应”

### 3.6 可信度与优先级

投资场景里，不能所有新闻都一样重。

每个事件建议增加：

- `source_priority`
- `officialness`
- `confidence`
- `impact_level`
- `price_sensitive`

示例：

- SEC 8-K + 财报指引下修
  高优先级，高可信度，高价格敏感
- 单家媒体转载的产品体验文章
  低优先级，低价格敏感

## 4. 产品层要补的功能

### 4.1 查询方式升级

从：

- 公司名 + 时间范围

升级为：

- 公司名 / Ticker
- 时间范围
- 市场
- 事件类型过滤
- 官方/媒体过滤
- 利多/利空过滤

### 4.2 结果页升级

当前主要是图。

目标结果页应至少包含：

1. 顶部研究摘要
- 关键结论
- 主要风险
- 主要利多

2. 时间线
- 按时间展示关键事件

3. 图谱
- 展示事件、主体、监管、合作方、披露之间的关系

4. 证据面板
- 每条结论都能点开看原文

5. 市场反应面板
- 股价和成交量联动

### 4.3 历史追踪

投资产品不能只做单次查询。

需要补：

- 公司 watchlist
- 自动日报/周报
- 新事件提醒
- 事件主线持续跟踪

## 5. 技术层要补的事情

### 5.1 存储层升级

当前：

- JSON 本地持久化

目标：

- PostgreSQL：任务、文章、事件、摘要
- Neo4j：图关系
- Redis：缓存

### 5.2 数据处理管道升级

建议拆成明确 pipeline：

1. 数据抓取
2. 清洗去重
3. 投资相关过滤
4. 事件抽取
5. 实体归一化
6. 事件聚合
7. 影响打标
8. 写入数据库和图数据库

### 5.3 评估体系

需要有内部评估，而不是只看“看起来像对了”。

建议指标：

- 投资相关率
- 事件抽取准确率
- 实体归一化准确率
- 同事件聚合准确率
- 利多/利空判断准确率
- 高价值事件召回率

## 6. 建议的演进路线

### Phase 1：投资研究增强版原型

目标：

- 明显区别于普通新闻工具

要做的事：

- 接入 SEC EDGAR
- 增加 ticker 输入
- 建立投资事件 schema
- 过滤非投资相关新闻
- AI 输出结构化投资事件
- 前端增加时间线与重点事件区

### Phase 2：投资情报图谱 MVP

目标：

- 可以持续使用的产品版本

要做的事：

- 接入金融新闻 API
- 接入价格数据
- 增加利多/利空和影响等级
- Neo4j 入库
- 图谱支持事件主线查看
- 支持 watchlist

### Phase 3：投资情报平台

目标：

- 从单次研究工具升级到持续情报系统

要做的事：

- 多公司对比
- 自动日报/周报
- 事件告警
- 分析师工作流
- 图谱问答
- 组合风险跟踪

## 7. 当前最值得先做的三件事

如果只看投入产出比，最值得先做的是：

1. `Ticker + 公司别名体系`
- 先解决 `Google / Alphabet`、`Meta / Facebook` 这类识别问题

2. `投资事件 schema`
- 把事件类型、影响方向、影响等级、证据结构固定下来

3. `价格反应联动`
- 让事件不只是新闻，而是能和市场反应一起看

## 8. 一句话结论

从“研究工具原型”进化到“投资情报图谱产品”，核心不是把图画得更复杂，而是：

- 用更高价值的数据源
- 建立真正服务投资决策的事件 schema
- 把新闻、披露、实体关系和市场反应组织成可追溯的结构

只有这样，这个产品才会明显区别于普通新闻聚合器。
